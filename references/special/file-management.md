# 书本内容分类管理系统（Project File Tree）

> 一本网文的完整生命周期涉及多种文件：设定、正文、存稿、已发布章节、工程管理、存档。本系统定义标准化的中文目录结构和 docx 文件管理规范。

## 一、标准目录结构

```
《书名》/                          ← 项目根目录，书名用中文全角书名号
│
├── 项目承接.md                    ← 新会话一键恢复（见 session-handoff.md）
├── 记忆锚点.md                    ← 全书压缩快照（见 memory-anchor.md）
│
├── 设定/                          ← 【设定层】所有背景设定文件
│   ├── 00_设定总纲.docx           ← 立项卡+金手指+卖点+差异化
│   ├── 01_人物档案.docx           ← 全员人设卡+语言指纹+情绪轨迹
│   ├── 02_大纲章纲.docx           ← 全书主线+分卷+章纲
│   ├── 03_伏笔追踪.docx           ← 伏笔编号+埋点+计划回收
│   └── 04_世界观与势力.docx       ← 力量体系+势力+政治+经济+地理+历史
│
├── 正文/                          ← 【写作层】正在撰写中的章节
│   ├── 第1卷_起始/                ← 每卷一个子目录
│   │   ├── 第001章.docx
│   │   ├── 第002章.docx
│   │   └── ...
│   ├── 第2卷_崛起/
│   └── ...
│
├── 存稿/                          ← 【缓冲层】写完待发、未发布的缓冲
│   ├── 第2卷_存稿/                ← 与正文对应，存未发布的章节
│   │   ├── 第041章.docx
│   │   └── ...
│   └── 待修改章节/                ← 写了但需要大改的
│
├── 已发布/                        ← 【发布层】已上传到平台的终版
│   ├── 番茄_已发布/               ← 按平台分目录
│   │   ├── 第1卷/
│   │   └── ...
│   └── 起点_已发布/               ← 如果一稿多投
│
├── 工程/                          ← 【管理层】项目管理与运营
│   ├── 进度日志.md                ← 每章写完更新
│   ├── 投稿记录.md                ← 每次投稿/签约记录
│   ├── 改稿记录.md                ← 修改了什么、为什么改
│   ├── 数据追踪.md                ← 收藏/追读/完读率变化
│   ├── AI协作记录.md              ← 每次有效 prompt/工作流记录
│   └── 番茄运营日志.md            ← 推荐位/首秀/活动记录
│
└── 存档/                          ← 【存档层】历史版本和废弃内容
    ├── 废稿/                      ← 废弃章节，不删（可能复用）
    ├── 旧版设定/                  ← 改过的设定版本
    └── 卷X_锚点快照/              ← 每卷结束时的状态快照
```

## 二、各文件的用途与格式

### 设定层（`设定/`）

| 文件 | 格式 | 内容 | 何时更新 |
|---|---|---|---|
| `00_设定总纲.docx` | docx | 立项卡、金手指、核心卖点、差异化、主线终局 | 开书时创建，大改时更新 |
| `01_人物档案.docx` | docx | 主角+反派+重要配角人设卡、语言指纹、情绪轨迹 | 新人登场时追加 |
| `02_大纲章纲.docx` | docx | 全书主线→分卷→前20章章纲 | 新卷开始前写好 |
| `03_伏笔追踪.docx` | docx | 伏笔编号表（F001-F999） | 埋新伏笔时追加，回收时标记 |
| `04_世界观与势力.docx` | docx | 力量体系、主要势力、政治经济、地理、历史大事 |强设定题材开书前搭好 |

### 写作层（`正文/`）

- 每卷一个子目录，命名：`第N卷_卷名`
- 每章一个 docx 文件，命名：`第NNN章.docx`（3位编号，对齐排序）
- 章节文件格式：标题用"第X章 章名"，正文宋体小四/五号，1.5倍行距

### 缓冲层（`存稿/`）

- 写完待发但还没发布的章节放这里
- 与正文目录结构对应
- `待修改章节/` 放写了但需要大改的，不删不混

### 发布层（`已发布/`）

- 按发布平台分目录
- 已发布 = 终版，不再修改（要改也是记到改稿记录里，从存稿出新版）
- 用于：核对已发内容的设定一致性、对比改版效果

### 管理层（`工程/`）

- `进度日志.md`：每章写完更新，记录日期、字数、完成状态
- `投稿记录.md`：按 `submission-guide.md` 的格式记录每次投稿
- `改稿记录.md`：记下"什么时候、改了什么、为什么改"，防止反复改动同一个地方
- `数据追踪.md`：记录平台的收藏/追读/完读数据变化趋势
- `AI协作记录.md`：记录每次有效的 prompt/工作流，形成可复用的操作手册
- `番茄运营日志.md`：番茄专属——首秀时间、推荐位、活动参与、数据波动

### 存档层（`存档/`）

- `废稿/`：废弃但可能复用的章节
- `旧版设定/`：改版前的设定快照
- `卷X_锚点快照/`：每卷结束时的 `记忆锚点.md` 快照——用于回溯

## 三、docx 文件规范

### 为什么用 docx 而非 md
- docx 是网文行业标准交换格式（编辑收稿、平台上传）
- 支持格式刷、批注、修订模式——改稿必备
- 可与 `documents` skill 配合生成排版精良的投稿稿

### docx 命名规则
- 中文命名，编号用3位数字对齐（第001章、第010章、第100章）
- 卷目录命名：`第N卷_卷名`（卷名可用2-4字概括，如"第1卷_起始""第2卷_崛起"）
- 不要用空格、特殊符号

### docx 内容格式
- 章节标题：宋体二号加粗居中
- 正文：宋体小四号，1.5倍行距
- 段落首行缩进2字符
- 场景切换用 `***` 或 `◇` 分隔

## 五、与 Codex 工作流的集成

### 开新书时
1. 在 `E:\ZhuoMian\` 下用初始化脚本创建项目目录
2. 在 Codex 中调用 documents skill 创建设定层的 docx 文件
3. 填写 `记忆锚点.md` 和 `项目承接.md`

### 日常写作时
1. Codex 读 `记忆锚点.md` → 读上章正文 → 写新章 → 保存到 `正文/`
2. 更新 `记忆锚点.md` 和 `工程/进度日志.md`
3. 积累到一定量 → 移到 `存稿/` → 发布后移到 `已发布/`

### 改稿时
1. 从 `已发布/` 或 `正文/` 取原文
2. 在 `存稿/待修改章节/` 存修改版
3. 改稿记录写到 `工程/改稿记录.md`
4. 定稿后覆盖到 `正文/` 对应位置

### 新会话承接时
1. Codex 读 `项目承接.md` → 读 `记忆锚点.md` → 继续工作
2. 不需要用户重新解释"我在写什么"

## 六、多平台一稿多投管理

如果同一本书投多个平台，`已发布/` 下按平台分：

```
已发布/
├── 番茄_已发布/          ← 番茄版（砍铺垫、快节奏）
├── 起点_已发布/          ← 起点版（补设定、长伏笔）
└── 晋江_已发布/          ← 晋江版（加情感线）
```

各平台版本独立维护，记录在 `工程/改稿记录.md` 中的平台差异。
